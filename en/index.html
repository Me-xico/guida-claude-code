    A Guide to Claude Code¶

    Claude Code is a coding agent that lives in the terminal: you describe an outcome ("the login form doesn't validate the email: find out why and fix it, with a test") and it reads the codebase, edits files, runs commands and tests, and iterates until the task is done, asking your permission for every action that matters. It's not a chat you copy code from, nor an autocomplete: it's a collaborator working in your repo, and this guide teaches you how to make it work well.

    Everything was written and checked in July 2026 against Claude Code 2.1.210. The commands were run for real, the screenshots come straight from the TUI, and the facts were verified against the official documentation — still the place to look once this guide starts to age.

    How to read it¶

    A note on numbering: the number in a chapter's name is a stable identifier, not the reading order — the guide grows by section and new chapters get appended at the end. The right order is this index.

    Basics - starting right (day 1):

    1. Installation and first run - from curl to your first prompt
    2. Setup and configuration - where config lives, permissions, deny rules for secrets
    3. Day-to-day use - the workflow, undo, session management
    4. CLAUDE.md and rules - the project's permanent instructions

    Method - what makes the difference:

    1. Verification - if you read only one chapter, make it this one
    2. Prompt engineering - asking for things in a way that works
    3. Research and design - deep research, SPEC, Ultraplan and Claude Design: decide well before you build
    4. Common mistakes - the checklist that closes the Method

    Power - once the basics are running:

    1. Skills and slash commands - reusable procedures
    2. Agents - who to delegate the noisy work to
    3. Hooks - rules that always hold, not almost always
    4. MCP - connecting browsers, Figma, GitHub
    5. Plugins - your setup in a package
    6. Frontend workflow - for frontend devs: Claude with a real browser

    Reference:

    1. Must-haves and costs - the first 30 minutes of setup, the plans, the money-saving habits
    2. Saving tokens - the tools for longer work windows on the Pro plan
    3. A real setup: global, per client, per repo - how the author actually organizes it, three levels at a time
    4. The setup in plain sight - the actual files line by line: permissions, hooks, agents, skills, MCP

    What you need to follow along¶

    A terminal, a Claude subscription (Pro is enough: ch. 01) and half an hour for the Basics section. Chapters 02 and 04 use a small React/TS project as their example; its files (CLAUDE.md and .claude/settings.json) are shown in full within the chapters themselves: there's nothing to download.
